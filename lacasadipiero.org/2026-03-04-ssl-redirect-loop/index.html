Problema risolto

Il sito era irraggiungibile a causa di un'impostazione sbagliata nella protezione SSL di Cloudflare. Corretta l'impostazione, il sito funziona regolarmente.

Il problema: il sito non si apriva

Aprendo lacasadipiero.org il browser mostrava un errore tipo "troppi reindirizzamenti" e la pagina non si caricava mai. Il sito risultava completamente inaccessibile per qualsiasi visitatore.

La causa: impostazione di sicurezza sbagliata

Il sito usa Cloudflare come "scudo" di protezione. Cloudflare era impostato per parlare con il server in modo non sicuro (HTTP), ma il server pretendeva una connessione sicura (HTTPS). I due si rimbalzavano la richiesta all'infinito, come due persone che si fanno "prima tu, no prima tu" alla porta.

Risolto: corretta la modalita di connessione

Abbiamo impostato Cloudflare per parlare con il server in modo sicuro (HTTPS), eliminando il rimbalzo infinito. Il sito si apre regolarmente.

$ curl -sI https://lacasadipiero.org/ HTTP/2 301 → Location: https://lacasadipiero.org/ (redirect loop) $ curl -sI http://127.0.0.1/ -H 'Host: lacasadipiero.org' (on origin) HTTP/1.1 301 → Cloudways maintenance-domain-mapping page (403) $ curl -sI http://127.0.0.1/ -H 'Host: lacasadipiero.org' -H 'X-Forwarded-Proto: https' HTTP/1.1 200 OK ← WordPress risponde correttamente con X-Forwarded-Proto Root cause: Cloudflare SSL mode = FLEXIBLE Origin ha certificato Let's Encrypt valido (exp 2026-04-13) WordPress siteurl = https://lacasadipiero.org → CF invia HTTP all'origin → WP redirect a HTTPS → CF ripete → loop

Fix applicato

# Cloudflare API — Zone lacasadipiero.org # SSL/TLS mode: flexible → full PATCH /zones/{zone_id}/settings/ssl { "value": "full" } # Verificato: origin cert CN=lacasadipiero.org valid until 2026-04-13 # Sito risponde 200 OK dopo il cambio

Note

  • Il certificato Let's Encrypt scade il 13 aprile 2026 — Cloudways dovrebbe rinnovarlo automaticamente
  • C'e anche un'app duplicata (djttgwsnfw) con lo stesso dominio nel conf — potrebbe essere un vecchio clone da verificare
  • "Always Use HTTPS" era OFF su Cloudflare — ora con SSL Full non serve, ma potrebbe essere abilitato per forzare HTTP→HTTPS client-side
